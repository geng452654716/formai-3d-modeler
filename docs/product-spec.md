# 产品规格（MVP）

## 1. 产品定位

这是一个面向 3D 打印的 AI 建模桌面软件。核心用户不需要掌握传统建模软件的完整操作体系，而是通过图片、自然语言、尺寸参数和局部选择来创建、检查和修改模型。

首版重点不是人物雕刻或动画，而是：

1. 产品外观与硬表面模型；
2. 电子产品外壳、保护壳和包裹件；
3. 能以毫米为单位精确制造的 3D 打印模型；
4. 在满足精确尺寸的前提下，逐步增加游戏/道具类多边形建模能力。

## 2. 已确认需求

### 2.1 输入

- 单张参考图片；
- 正面、侧面、背面等多视图图片；
- 对电子开发板拍照，自动识别 USB、供电、按键、指示灯等需要暴露的接口位置；
- 自然语言建模与修改指令；
- 关键物理尺寸；
- 电子元件的长、宽、高、位置和安装方向；
- 后续支持导入电子元件的 STEP、STL 或 3MF 模型。

单张图片生成完整 360° 模型时，未显示区域只能由 AI 推测。精确制造至少需要用户在图片上选择两个标定点，并输入两点之间的真实尺寸，由系统计算像素距离和毫米/像素比例；电子外壳类模型优先要求多视图或元件尺寸。第一版多视角联合标定使用各视角尺度中位数和最大偏差提示，不等同于摄影测量相机位姿求解。

接口识别结果必须包含位置、估算宽高、置信度、是否需要开孔和可选二维开孔轮廓，并与精确 CAD 特征分开标记；不同视角的接口对应必须区分“已匹配”和“待确认”。待确认接口必须由用户确认、拆分或忽略后才能应用联合结果；用户可修正接口 ID、类型、所在面、尺寸、偏移、开孔要求和开孔轮廓，修正值优先于单图识别估算。单视角识别完成后，用户还可在已标定二维照片平面上直接选择、拖动和缩放接口框；系统按毫米/像素比例回算接口宽高、以图片中心估算横向偏移、以接口框底边估算底部偏移。照片框编辑不是相机位姿求解，修改后必须重新执行多视角联合标定。

### 2.2 建模能力

- 创建基础几何体和硬表面模型；
- 精确移动、旋转和缩放；
- 实体并集、差集、交集；
- 内部掏空和指定壁厚；
- 按电子元件尺寸生成包络空间；
- 为元件设置装配间隙；
- 创建开孔、线缆出口、散热孔、螺丝孔和螺丝柱；
- 倒角和圆角；
- 外壳分件、上盖/下盖以及装配结构；
- 颜色调整；
- 后续增加自由曲面和有机网格建模。

### 2.3 交互能力

- 鼠标旋转、缩放和平移视角；
- 点击选择对象或部件；
- 使用三维操控器移动、旋转和缩放；
- 框选或点击局部区域后向 AI 下达指令；参数化精确 CAD 已支持点击单个稳定面、点击单条稳定边、按顺序手工多选 2–64 条稳定边链，并框选当前视角射线可见的多个稳定面；
- 框选先执行屏幕空间三角形与矩形相交检测，再用摄像机射线第一命中过滤隐藏面，避免把被遮挡的背面一并提交；
- 截取局部视图并附加给 AI；稳定面 ID、原始毫米坐标、逆变换后的外法线、零件包围盒、摄像机矩阵和尺寸上下文与 PNG 截图一起提交；
- 手工编辑顶点、边和面；
- 撤销、重做；
- 模型版本历史；
- 两个版本的重叠、并排和差异对比；
- 导出 STL 和 3MF。

### 2.4 AI 行为

- 复用本机 Codex 的安装和现有 API Key 登录状态，不要求用户在本产品界面重复填写密钥；
- Codex 自动执行修改；
- 每次 AI 修改前必须创建版本快照；
- 执行失败时不覆盖最后一个有效版本；
- 修改完成后自动运行几何和打印检查；
- 用户可以查看 AI 当前阶段、工具调用和结果摘要；
- 用户可以从历史版本恢复或从任意版本创建分支。

当前交付状态：已支持历史列表、任意参数化版本恢复和从恢复节点继续修改形成分支，并可比较全部参数与通用开孔元数据。存在本机精确快照时，可加载历史 STL 与当前精确 CAD 做半透明重叠和按真实毫米尺寸并排视觉比较，也可按稳定零件 ID 对历史与当前 STEP 执行 OpenCascade 双向精确布尔差集，以绿色/红色高亮新增和删除区域并报告体积。参数化 CAD 清单现已记录“几何签名匹配第一版”的面描述和稳定 ID，版本实体对比可统计共享、新增与消失编号；精确 CAD 的专用选择 STL 还会把连续 `triangleIndex` 区间回映射到这些稳定面，支持点击单面和框选多个当前可见面。该能力不是 OpenCascade 原生永久拓扑命名，`triangleIndex` 只对同一次生成的选择网格有效，大幅拓扑变化、对称面或布尔重建仍可能重新编号。上传 STL 使用独立网格区域/壁厚采样协议，不伪装成参数化 CAD 稳定面选择；界面必须明确区分元数据差异、STL 视觉比较、实体级精确布尔差异和第一版稳定面选择。

稳定 CAD 面局部特征第一版已经接入：用户点击单个稳定平面并输入包含直径或半径、以及高度或深度的中文指令后，可生成圆形凸台或切除圆孔。Codex 登录时只能返回受限的 `add-cylinder | cut-cylinder` 结构化计划，不能输出或执行任意 Python、CadQuery、shell，也不能在同一计划中同时修改整模参数；计划中的 `partId + stableFaceId` 必须逐字绑定当前点击选择，最终点击中心与法线只取自前端已验证上下文。Codex 不可用时才使用确定性中文尺寸解析器。Worker 必须从当前 STEP 重新定位 OpenCascade 面并使用真实外法线执行布尔；视口点击法线只做点积一致性检查。结果必须通过有效性、封闭性、单一 Solid、体积方向、点击距离、法线方向和 STL 再导入检查，之后重新生成面签名、选择 STL、面映射和 3MF。修改后原 `triangleIndex` 立即失效。`localFeatures[]` 会在后续参数化整模重建时从基础参数化实体开始按顺序安全重放，并使用每条记录的 `targetFace` 几何签名快照辅助重新定位；稳定面、记录中心、外法线、尺寸或布尔结果任一不一致时，必须拒绝整次重建并保留最后有效模型，不能静默跳过。第一版仍不支持框选多面、曲面 UV、矩形孔/槽孔、整面拉伸、局部圆角/倒角或任意拓扑编辑。


### 2.5 拆件、连接与紧固结构

- 提供装配视图和拆分视图，装配视图默认显示最终闭合效果；
- 自动拆件支持适配打印空间、最少支撑和手工分型面三种策略；
- 连接结构首批支持圆柱定位销、D 形防转销、燕尾榫、球头和磁铁孔；
- 紧固结构支持 PLA/PETG 卡扣、M2/M2.5/M3 螺丝柱、通孔、沉孔和沉头孔；
- 螺纹提供两个独立选项：M2/M2.5/M3 打印友好圆脊近似内外螺纹，以及已经实体化的 M2/M2.5/M3 ISO 公制粗牙 60° 内外螺纹；界面必须明确区分两种牙型；
- 所有公母连接结构必须提供独立间隙参数，并按材料和喷嘴给出过紧、薄壁与断裂风险提示；
- 手办和任意网格模型需要增加曲面切割、连通区域分析、薄弱区域分析、非流形修复和打印方向评估。

当前迭代已完成制造方案规划器、通用 CAD/上传 STL 的 OpenCascade 平面拆件与封闭补面、体积守恒验证、桌面命令接入、视口装配/拆分显示和 STEP/STL 导出。圆柱定位销、D 形防转销、燕尾榫、球头/球窝、磁铁孔，PLA/PETG 可拆悬臂卡扣，M2/M2.5/M3 螺丝柱、底孔、通孔和圆柱沉孔，以及两类 M2/M2.5/M3 内外螺纹均已经由 OpenCascade 布尔写入精确实体。打印友好近似螺纹使用圆脊截面，面向 FDM 直接打印装配；ISO 螺纹按 ISO 261 粗牙系列使用 M2×0.40、M2.5×0.45、M3×0.50 毫米螺距，并沿螺旋扫掠 ISO 基本 60° 梯形截面。两类牙型在界面和协议中是独立选项，不能互相冒充。

ISO 内外螺纹使用同一螺旋起点和相位，沿 X/Y/Z 任意切割轴的局部 U/V 坐标生成；结果返回公称直径、螺距、有效长度、60° 牙型角和“ISO 公制粗牙基本牙型”标准名称。考虑 P1S、0.4 毫米喷嘴和 PLA/PETG 的实际打印偏差，配合内螺纹孔仍加入独立径向补偿和受限轴向补偿；该补偿属于 FDM 制造适配，不改变 ISO 基本牙型选项的标识。生成过程会检查实体有效性、封闭性、单一 Solid、连接结构最小设计壁厚、公母间隙和装配干涉。`minimumDesignedWallMm` 只校验新生成连接结构，不得与全模型壁厚采样混用；全模型结果由独立壁厚分析协议提供。界面必须继续明确区分“识别结果”“已规划”和“已生成精确实体”。

### 2.6 全模型壁厚分析

- 对任意封闭 CAD STEP/STP 零件和用户上传 STL 执行全局表面采样，不绑定固定模型类别。
- 默认按 P1S、0.4 毫米喷嘴和 PLA/PETG 设计检查分为四级：危险 `< 0.80` 毫米、偏薄 `0.80–1.20` 毫米、建议 `1.20–2.00` 毫米、充足 `>= 2.00` 毫米。
- 界面必须显示最薄值、5% 分位、有效样本数、覆盖率、四级样本计数和三维红/橙/黄/绿热力图。
- 结果名称必须明确为“表面法向射线采样估算”；不得宣传为有限元、受力、疲劳寿命或连续精确壁厚场。
- 覆盖率过低、复杂非流形、自相交或无法形成有效封闭实体时必须中文拒绝，不得输出伪造的安全结论。
- 模型几何、版本、来源或拆件状态改变后，旧壁厚结果必须失效；CAD 与上传 STL 之间不得交叉复用热力图。
- 最近样本颜色只用于显示离散风险，不代表每个顶点都完成了独立精确射线求交。
- 热力图开启后，用户可点击模型表面选择最近的离散采样区域，或一键定位最薄处；界面显示三维标记、原始毫米坐标、估算壁厚和风险等级。
- CAD 局部选择会作为明确几何上下文附加给 Codex；参数化修改完成后，系统对同一零件重新执行壁厚采样，并按原坐标寻找最近采样点，报告修改前后的估算壁厚变化。
- 上传封闭单 Solid STL 已建立第一版受限局部实体重建：输入来自壁厚离散样本的原始毫米坐标和单位内法向，支持沿外法向增加圆形凸台，以及沿内法向切除圆孔。
- 上传 STL 局部修改后必须基于当前工作文件重新执行“表面法向射线采样估算”，并保留原始上传 STL；该能力不等同于参数化特征树、任意网格编辑、自由曲面重建或多 Solid 局部编辑。

### 2.7 照片精确开孔与制造校验

- 经过人工复核且 `requiresOpening=true` 的接口，在能够确定正交接口面后进入通用精确开孔协议；支持正面、背面、左侧、右侧、顶部和底部。
- 二维开孔轮廓支持圆孔、矩形孔、圆角矩形孔和槽孔；用户可人工修改接口类型、所在面、宽高、位置、是否开孔和轮廓。
- 开孔坐标来自已标定的二维照片平面。单张透视图不能单独确定精确接口面，也不能提供相机位姿、摄影测量或任意物体三维重建结果。
- OpenCascade 对顶部开孔切削上盖，对其余五面开孔切削主体；每次布尔运算后必须检查实际相交、实体有效性、封闭性和单一 Solid。
- 生成前检查保守矩形包围盒的边缘余量和同面孔间距；这些检查不是圆角边界精确应力分析、受力分析、疲劳分析或有限元分析。
- 新照片开孔保存相对接口面水平中心和底边的毫米定位锚点；之后改变外壳长、宽、高时，系统按当前接口面尺寸重新计算中心坐标，但开孔宽高、水平偏移和底部偏移不会按比例缩放。
- 老项目没有照片定位锚点元数据时继续使用原固定毫米坐标，并在界面提示重新复核；自动重算不能替代卡尺确认。
- 照片与 AI 尺寸只能作为建模输入，制造前必须使用卡尺复核关键孔位、孔径、板件包络和装配间隙。

## 3. 精度与单位

- 项目内部统一使用毫米；
- 视口、参数输入、导入和导出都必须明确单位；
- STL 本身不保存单位，因此导出元数据和导入流程必须固定按毫米解释；
- 每个项目保存打印机、喷嘴、材料、默认间隙和最小壁厚配置；
- 模型导出前检查包围盒是否超出打印机成型尺寸。

首版默认打印配置：

- 打印机：Bambu Lab P1S；
- 喷嘴：0.4 mm；
- 材料：PLA、PETG；
- 默认外壳壁厚：2.0 mm（可修改）；
- 默认 XY 装配间隙：0.30 mm（可修改）；
- 默认 Z 装配间隙：0.50 mm（可修改）。

## 4. 电子元件包裹工作流

1. 用户创建或导入电子元件；
2. 为每个元件记录精确尺寸、禁入空间和接口位置；
3. 用户输入外壳风格、壁厚、间隙和开合方式；
4. AI 生成元件包络体；
5. AI 在包络体外生成外壳实体；
6. AI 创建内部空腔、安装柱、定位槽和接口开孔；
7. 软件检查元件碰撞、最小间隙、最小壁厚和封闭性；
8. 用户通过局部选择继续修改；
9. 导出 STL 或 3MF，并交给拓竹切片软件。

## 5. 参数化与网格编辑的边界

精确 CAD 实体与自由网格编辑是两种不同的数据模型：

- 参数化实体适合尺寸、壁厚、圆角、孔位、外壳和装配；
- 网格编辑适合直接修改顶点、边、面和有机外形。

首版将参数化实体作为模型源数据。用户进入顶点/边/面编辑时，系统创建一个“网格分支”，保留原参数化版本。网格分支可以继续打印和局部编辑，但不能保证所有 CAD 特征仍可无损参数化修改。

## 6. 导出范围

### STL

- 输出毫米尺度的三角网格；
- 支持单零件或按部件分别导出；
- 导出前检查封闭、流形、自交、法线和退化三角形。

### 3MF

目标为拓竹工程 3MF，除几何、单位、对象层级、变换和基础颜色外，还需要保存 P1S 打印机配置、热床摆盘、材料、层高、填充和支撑等切片工程信息。实施时先完成几何 3MF，再逐步补齐并验证 Bambu Studio 工程兼容性。

## 7. MVP 验收案例

首个端到端案例使用 ESP32-S3 N16R8 42 针开发板外壳：

1. 拍摄开发板正反面及接口侧照片，自动识别 PCB、排针、USB 接口、按钮和指示灯；
2. 用户确认识别结果，并提供至少一个真实尺寸用于照片尺度校准；
3. 生成带指定壁厚和装配间隙的上下壳；
4. 自动创建内部空腔、PCB 定位结构和接口开孔；
5. 用户说“顶盖增加散热孔，四角更圆，接口孔右移 1.5 mm”；
6. Codex 自动修改并保存新版本；
7. 视口显示前后版本差异；
8. 验证模型尺寸和封闭性；
9. 导出 STL 和适用于 P1S 的拓竹工程 3MF。


### 37. 手工多选边链圆角与倒角第一版（已实现）

- 用户可切换“多选边链”，在任意参数化 CAD 零件上按顺序选择 2–64 条稳定边；视口以中文显示已选边数、零件数和精确解析状态。
- 每条边必须完成当前修订的 OpenCascade 精确解析，逐边保存所属稳定面、稳定边、真实毫米点击点、外法线和 UV。不同所属稳定面可以加入同一边链，但不同零件不能混选。
- 支持无分叉开放链与闭合链；拒绝重复物理边、不连续边、分叉拓扑、退化边、少于 2 条和超过 64 条的选择。系统不会自动添加用户未选择的边。
- 中文指令可对整条手工边链执行统一半径圆角或统一距离倒角。Codex 只能返回受限结构化计划，逐边列表必须与用户选择完全同序，不能增删、换序、替换或退化为单边操作。
- 成功后清除已失效的边链选择并报告实际处理边数；Worker 或重放失败时保留最后有效模型，当前执行失败时还要保留全部手工选择以便用户调整后重试。
- 版本记录保存逐边稳定 ID、所属面和几何签名快照；整模重建时逐边重新定位和刷新快照，任一目标失配都必须原子拒绝，不能只重放部分边。

## 连续共面区域边界线框与重复选面性能

第 47 阶段把执行前预览从“只看黄色面片”扩展为“黄色区域面 + 青绿色闭合边界环”。用户可以直接观察区域外边界和现有孔洞，确认连续共面扩展是否越过了预期范围；所有线框点都以源毫米坐标保存，并在渲染时才转换到当前装配或拆分视图。第一版对外环和孔洞环使用同一种颜色与说明，不承诺数组顺序、环方向或语义名称。

同一模型修订第一次点击种子面时构建全网格三角面索引和共享边拓扑，之后切换种子面复用缓存，不再反复扫描整张大网格。模型修订、重新上传、版本恢复、几何或视图来源变化后缓存必须失效。该优化只改善前端交互延迟，不放宽 20,000 个三角面、200,000 平方毫米等安全上限，也不替代桌面 Worker 对当前工作 STL 的独立复核。

下一阶段将基于种子平面的二维投影和环包含关系区分外环与孔洞，并显示外环/孔洞数量、各环周长与包围尺寸；不得仅依赖 STL 三角面绕序或有符号面积方向判断语义。第一版不扩展到曲面区域、嵌套岛、多区域框选或自由编辑。

## 连续共面区域外环/孔洞语义与尺寸测量

第 48 阶段把边界线框升级为可读的环语义和尺寸反馈。系统将全部闭合环投影到种子平面二维坐标，使用环包含关系计算嵌套深度：未被其他环包含的是外环，被一个外环包含的是孔洞。判断不得依赖 STL 三角面绕序、边界环数组顺序或单纯的有符号面积方向。

预览卡必须分别显示外环和孔洞数量，并为每个环显示真实三维周长以及种子平面中的二维包围宽度和高度。视口继续以黄色显示完整区域，以青绿色显示外环、以珊瑚色显示孔洞环，并提供全中文图例。模型修订、导入、恢复、选择或操作模式变化后，旧分类和测量必须与区域预览一起失效。

前端 `boundaryLoops` 只属于当前修订的执行前反馈，保存源毫米坐标、`outer|hole` 语义、周长、包围尺寸和嵌套深度；兼容的 `boundaryLoopsMm` 继续保留。桌面 Worker 不接收、不读取也不信任前端分类，正式执行时仍从当前工作 STL 独立扩展区域并由 OpenCascade 判断建模 Wire 语义。第一版检测到嵌套岛时安全拒绝，不扩展到曲面区域、多区域框选或自由编辑。

下一阶段将实现边界环点击聚焦与视口尺寸标注第一版：用户可从测量列表选择某个外环或孔洞，视口只强化当前环并显示周长、包围宽度和高度标注，便于在多个孔洞间定位。聚焦状态只用于查看，不改变 Worker 请求、区域扩展或建模语义；第一版不提供拖动尺寸、直接编辑边界或曲面标注。

# FormAI 实施计划

更新日期：2026-07-20

## 状态定义

- **已实现**：已经存在可运行代码，并通过自动化或人工检查。
- **本阶段已完成**：本轮要求已写入真实数据流或 OpenCascade 实体，并完成验证。
- **已规划**：界面和参数方案已经存在，但尚未写入精确实体。
- **未实现**：尚无可用代码，不能在界面中宣称已经完成。

- React Three Fiber 三维视口、装配视图、拆分视图、对象选择和基础参数面板。
- CadQuery/OpenCascade 通用模型 Worker、动态零件清单，以及当前电子元件保护壳模板的内部掏空、圆角、倒角、接口开孔和独立上盖。

- 拆件策略、连接结构、卡扣、螺丝和螺纹的制造方案规划器。

## 本阶段已完成

### 1. 照片输入与双点尺寸标定

- 在原始照片上选择两个标定点并显示标定线。
- 读取图片原始像素尺寸，计算标定线像素长度和毫米/像素比例。
- 未选满两个点时禁止执行“精确标定识别”。
- 将标定点、像素长度、真实长度和毫米/像素比例传给 Codex。

验收标准：识别请求中包含完整标定数据，界面明确显示像素距离和比例，而不是只有一个无对应位置的尺寸输入框。

### 2. 元件接口位置自动识别

- 识别 USB-C、按钮、LED、排针、电源接口及未知接口。
- 返回接口中心位置、估算宽高、置信度和是否需要外壳开孔。
- 在照片上叠加中文接口标记。
- 将参考照片元数据和识别接口持久化到项目 Store。
- 把经过人工复核、需要开孔且能够确定正交接口面的识别结果转换为通用精确开孔记录，不绑定 USB-C 或固定模型类别。
- 视觉结果包含可选二维开孔轮廓；不确定时使用圆角矩形并给出中文复核警告。

验收标准：应用识别结果后，接口类型、所在面、轮廓、宽高和二维标定位置进入通用开孔协议，并触发精确 CAD 重建；透视图不能单独决定精确接口面。

### 3. OpenCascade 平面拆件与自动补面

- 对封闭 CAD 实体使用切割平面两侧的大包围盒执行 Boolean Common。
- OpenCascade 布尔结果必须形成两个封闭 Solid，切割面由内核自动补面。
- 检查两部分有效性、封闭性、正体积和切割前后体积守恒。
- 提供 STEP/STL 验证输出和自动化测试脚本。
- 已接入 Tauri/Rust 命令与中文“拆件与连接”界面，可从当前模型清单动态选择任意可拆零件，并设置 X/Y/Z 切割轴和毫米坐标。
- 精确拆件结果会替换视口中的原零件；“装配视图”合并显示，“拆分视图”沿切割方向展开。
- 带连接结构的两侧 STL/STEP 已加入导出菜单；桌面发布资源包含拆件 Worker 和精确连接特征模块。

验收标准：示例实体切割后两部分均为有效封闭实体，体积和误差在容差内；界面返回补面数、封闭性和体积守恒结果，并可查看和导出真实拆件。

### 4. 用户自定义 STL 通用拆件第一阶段

- 支持上传任意文件名、任意模型类别的二进制或 ASCII STL，不绑定电子外壳、开发板、主体、上盖或手办等固定语义。
- Web 与 Tauri 使用统一上传接口，限制 STL 格式、空文件和 50 MB 大小，并以安全内部文件名保存；界面和元数据保留原始文件名。
- 使用 OpenCascade 导入三角面、缝合 Shell，并将封闭流形网格转换为分面 Solid；开放网格或无法形成有效实体时返回中文错误，不输出伪造的可打印结果。
- 支持沿任意 X/Y/Z 轴和毫米坐标拆分；由 OpenCascade Boolean 自动生成两侧切割补面。
- 自动检查实体数、封闭性、正体积、补面数量和拆件前后体积守恒。
- 输出拆件两侧的 STEP/STL；上传原模型和拆件结果均可在视口中查看，并支持装配视图与拆分视图。
- 场景树、项目标题、状态栏和导出菜单根据用户上传模型动态生成，不出现固定模型部件命名。
- 上传模型激活时，右侧参数区切换为通用 STL 的文件名、尺寸、三角面、实体数、体积、封闭性和拆件校验信息，不再显示示例电子外壳的固定参数。
- 上传模型的候选连接位按真实包围盒和当前切割平面坐标轴计算，不使用默认开发板或壳体尺寸；生成时由精确内核使用真实实体相交体积重新选择能够同时附着到两侧拆件的位置。

验收标准：至少覆盖一个非项目内置的封闭 STL、一个二进制或 ASCII STL 以及一个开放 STL；有效模型可完成上传、预览、拆件、自动补面和导出，开放模型必须被明确拒绝。

### 5. 高级 STL/手办网格修复与自动拆件

**第一批已完成：**

- 通用读取二进制和 ASCII STL，按毫米公差焊接重复顶点并建立三角形边拓扑。
- 检测并清理退化三角形和重复三角形；统计边界边、非流形边和连通网格数量。
- 收集不分叉的开放边界闭环，校验法线方向、共面性、投影自交和嵌套轮廓。
- 对独立、简单、近似共面的孔洞执行二维投影和耳切三角化，生成修复后 STL 工作副本。
- 修复后再次检查开放边和非流形边，再交给 OpenCascade 缝合为封闭 Solid；原始 STL 始终保留。
- 上传清单和中文界面展示输入/输出三角面、清理数量、修洞数量、补面三角形及修复前后开放边。
- 修复后的工作 STL 用于视口和后续拆件，并可与原始上传 STL 分别导出。

**后续仍需实现：**

- 修复非流形边、自相交、复杂非共面破面、分叉/嵌套边界和细小孤岛。
- 对超大三角网格进行分块、简化、缓存和后台任务优化。
- 自动评估打印方向、薄弱区域、支撑成本与候选拆件位置。
- 根据模型局部形状自动建议拆件面和连接结构位置，而不是依赖固定模板。

### 6. 连接与紧固结构写入精确实体

**本批已完成：**

- 圆柱定位销、D 形防转销、燕尾榫、球头/球窝和双侧磁铁安装孔已作为 OpenCascade 加料/减料特征写入最终拆件。
- M2、M2.5、M3 螺丝柱、底孔、通孔和圆柱沉孔已写入最终拆件，并输出带结构的 STEP/STL。
- PLA/PETG 可拆悬臂卡扣已写入精确实体：负方向拆件生成 6.0 毫米宽、1.6 毫米臂厚的悬臂和单侧导入斜面扣钩，正方向拆件生成带间隙的臂槽、扣合窗口及反向弯曲释放空间。
- 卡扣和配合槽使用切割轴局部 U/V 坐标生成，已覆盖 X/Y/Z 三个切割轴，不绑定电子外壳、手办或其他固定模型类别。
- M2、M2.5、M3 打印友好圆脊近似内外螺纹已写入精确实体。内螺纹方案在负方向拆件生成带螺旋槽的螺丝柱，并在正方向拆件生成通孔和圆柱沉孔；外螺纹方案在负方向拆件生成一体式芯柱和圆脊螺旋，在正方向拆件生成带径向打印补偿的配合内螺纹孔。
- M2、M2.5、M3 ISO 公制粗牙 60° 内外螺纹已写入精确实体；按 ISO 261 粗牙系列分别使用 0.40、0.45、0.50 毫米螺距，并与打印友好圆脊近似牙型保持两个独立选项。
- ISO 牙型沿螺旋扫掠 60° 梯形截面，基本三角形高度为 `H = √3 / 2 × P`，工作牙高为 `5H/8`，牙顶截平宽度为 `P/8`，牙根宽度为 `3P/4`；内外螺纹使用同一螺旋起点和相位。
- 近似螺纹和 ISO 螺纹均使用切割轴局部 U/V 坐标覆盖 X/Y/Z 三轴，并返回公称直径、螺距和有效长度；ISO 结果额外返回 60° 牙型角和标准名称。配合孔按 FDM 打印加入独立径向补偿和受限轴向补偿，避免细螺距牙槽重叠。
- 根据切割轴建立局部平面坐标，使用真实支撑柱探针与两侧拆件的相交体积筛选可附着位置；默认目标为 2 个连接器和 2 个紧固结构，窄截面允许降为 2 个连接器和 1 个螺丝柱、卡扣、近似螺纹或 ISO 螺纹。
- 生成后检查 OpenCascade 实体有效性、封闭性、额外悬浮 Solid、连接结构最小设计壁厚、公母间隙和装配干涉。
- 公母间隙限制为 0.10–1.00 毫米；连接结构最小设计壁厚限制为 1.20 毫米，适配 P1S、0.4 毫米喷嘴及 PLA/PETG 的当前默认制造配置。
- 中文界面显示实际连接位置、连接器及螺丝柱/卡扣/近似螺纹/ISO 螺纹数量、卡扣长宽高、螺纹直径、螺距、有效长度、牙型角和标准名称、最小设计壁厚、装配干涉、补面数量和拆件体积误差；装配视图可查看闭合结果，拆分视图可查看两侧结构。

**下一批仍需实现：**

- 曲率、受力、疲劳寿命和打印方向分析；当前 `minimumDesignedWallMm` 仍只校验新生成连接结构自身的参数化设计壁厚，与下面的全模型表面采样结果是两个独立指标。

### 7. 全局壁厚采样与三维热力图

- 对任意封闭 STEP/STP 或上传 STL 复用通用实体导入、简单孔洞修复和封闭性校验，不绑定电子外壳、手办或固定零件类别。
- 对 OpenCascade 三角化表面按面积确定性采样，使用实体内部分类确定内法向，再沿内法向寻找最近下一处实体边界。
- 默认风险阈值为：危险 `< 0.80` 毫米、偏薄 `0.80–1.20` 毫米、建议 `1.20–2.00` 毫米、充足 `>= 2.00` 毫米。
- 返回最薄值、5% 分位、中位数、最大值、四级计数、有效采样数、表面积、三角面数和覆盖率；覆盖率低于 35% 时中文拒绝，避免把异常网格报告为安全。
- 三维视口使用原始 STL 坐标匹配离散采样点，并以红、橙、黄、绿顶点色显示最近样本风险；可关闭或重新显示热力图。
- 热力图支持点击局部区域和一键定位最薄处，三维视口显示发光标记及中文壁厚标签；Codex 面板同步显示来源零件、原始毫米坐标、风险等级和估算壁厚。
- CAD 参数化修改成功后，自动对同一零件重新执行壁厚采样，并按原始坐标恢复最近采样点，报告修改前后的局部估算壁厚变化。上传封闭单 Solid STL 已支持第一版受限局部实体重建：沿壁厚采样点的表面法向增加圆形凸台，或沿内法向切除圆孔；完成后读取当前工作文件重新采样。
- Web、Tauri/Rust 和 Zustand 数据流均已接入；参数修改、撤销/重做、新建项目、上传/移除 STL、拆件和 CAD 重建会清除旧分析，避免跨版本或跨来源复用结果。
- 该能力只能称为“表面法向射线采样估算”，不是有限元、受力或疲劳仿真；离散采样也不能保证覆盖未采样的极细局部。

验收标准：当前 CAD 零件和上传封闭 STL 均可完成分析；热力图、中文图例和统计与结果协议一致；可定位和清除局部采样点；CAD 参数化修改后能自动复查同一局部；上传封闭单 Solid STL 可连续执行圆形凸台加厚和圆孔切除并自动复查；切换来源或改变几何后不会继续显示旧结果。

### 8. 多视角联合标定（第一版已实现）

- 同一项目支持正面、背面、左侧、右侧、顶部、底部和透视照片；可一次多选，也可在向导中继续添加或移除照片。
- 每个视角独立保存文件名、视角、双点标定、导入时间、Codex 单图识别结果和接口观测；元数据及联合结果保存到本机应用存储，图片二进制暂不保存。
- 至少两个已完成标定和识别的视角才能执行联合标定；采用各视角毫米/像素比例的中位数作为联合尺度，并计算最大相对偏差。
- 最大尺度偏差超过 10% 时显示中文警告，要求检查共面关系并用卡尺复核，不自动声称制造级一致。
- 接口优先按跨视角稳定 ID 合并为“已匹配”；不同 ID 但类型和尺寸接近时只形成“待确认”候选；单视角接口继续保留。
- 多视角对同一 CAD 参数的估算值使用中位数融合；经过复核、需要开孔且可确定正交接口面的接口进入通用精确开孔协议，其他接口继续作为识别和避让信息保存。
- 当前实现复用桌面后端逐张调用 Codex 视觉分析，再在前端执行确定性的尺度融合与候选匹配；没有实现相机内外参求解、SfM、稠密摄影测量或任意物体三维网格重建。

验收标准：可管理至少两个不同视角并分别标定、识别；联合结果显示融合尺度、最大偏差、已匹配和待确认接口；尺度冲突有中文警告；应用后尺寸融合和通用正交面开孔规则可验证；所有能力边界在界面中可见。


### 9. 多视角接口人工确认与编辑闭环（第一版已实现）

- 联合标定结果中的待确认接口不能再直接应用；必须逐组执行“确认为同一接口”“拆为独立接口”或“忽略误识别”。
- 单视角接口可人工确认保留；尺寸相似但 ID 不同的跨视角候选可拆回各自独立接口，拆分后仍需逐一确认。
- 支持编辑接口唯一 ID、类型、所在面、宽高、照片横纵位置、物理横向偏移、底部偏移、是否需要开孔和开孔轮廓。
- 人工编辑值保存在联合结果的复核字段中，不覆盖原始 Codex 观测；项目接口列表和通用精确开孔协议优先使用复核值。
- 误识别接口可忽略并恢复；忽略项不会进入项目接口列表，也不会参与自动开孔。
- 联合结果和人工复核状态继续通过项目 Store 与本机存储保存；所有按钮、校验错误和状态提示均为中文。

验收标准：待确认项存在时“应用联合结果并生成精确模型”保持禁用；确认、拆分、忽略和编辑均有纯函数测试；全部待确认项处理完成后才能应用，且人工修正值进入接口输出和通用精确开孔映射。

### 10. 照片接口区域直接编辑（第一版已实现）

- 照片预览新增“尺寸标定 / 编辑接口”中文模式切换；识别前继续使用双点标定，识别后可进入接口直接编辑。
- 已识别接口按标定比例显示为真实宽高对应的矩形框；可点击选中、拖动修改中心位置，并通过右下角手柄调整宽度和高度。
- 界面区分原始识别、人工修改和当前选中状态，并显示接口类型、毫米尺寸、开孔要求、照片中心位置、横向偏移和底部偏移。
- 接口框中心和尺寸被限制在照片边界内，最小可操作宽高为 8 像素；正面接口横向偏移以照片中心为零，底部偏移按接口框底边到照片底边的已标定距离估算。
- 人工拖动或调整尺寸会写回当前单视角识别记录，并清空旧联合标定和跨视角匹配；界面用中文要求重新执行联合标定。
- 支持“恢复本次识别值”，恢复当前 Codex 识别批次的原始接口位置和尺寸。
- 当前能力只是在已标定二维照片平面上做确定性换算，不包含相机位姿优化、SfM、摄影测量或三维网格重建。

验收标准：毫米尺寸与照片百分比框双向换算正确；拖动和缩放不会越过照片边界；偏移同步更新；无效标定安全拒绝；接口编辑后旧联合结果不可继续应用。

### 11. 通用六面精确开孔（已实现）

- `InterfaceOpeningSpec`、Vite 开发后端、Tauri/Rust 后端和 Python Worker 已形成完整协议；前后端执行相同的数量、枚举、尺寸、置信度和字符串长度校验，并转换为 Python snake_case 字段。

## 不能混淆的能力边界

- **识别结果**：Codex 从图片返回的结构化估计，不等于已经生成 CAD 特征。
- **制造方案**：拆件、连接和紧固结构的参数规划，不等于已经写入实体。
- **精确实体**：只有经过 OpenCascade 重建、有效性检查和尺寸检查的结果才能标记为已生成。
- **上传网格修洞**：只对独立、简单、近似共面的开放边界做三角化补面；非流形、自相交、分叉、嵌套和明显非共面破面仍会中文拒绝。
- **拆件切割补面**：模型成为封闭 Solid 后，由 OpenCascade Boolean 为拆件两侧生成切割面；它和上传阶段的网格修洞是两套不同校验过程。
- **全模型壁厚采样**：对原始完整实体执行离散法向射线估算；它不等于连接结构的 `minimumDesignedWallMm`，也不等于受力、疲劳或有限元仿真。

### 20. 稳定 CAD 平面矩形与槽孔局部特征（已实现）

- 通用平面轮廓协议扩展为五种受限操作：圆形凸台、圆孔、矩形凸台、矩形孔和槽孔；不绑定 ESP32、电子外壳、手办或固定模型类别。

### 26. 曲面圆形作用区域的自交和相邻面干涉检查（已实现）

- OpenCascade Worker 在执行曲面 `add-cylinder` 或 `cut-cylinder` 的布尔运算和任何 STEP/STL/3MF 文件写回前，逐面计算圆形工具体与修改前源实体稳定面的真实交集，不再只依赖布尔后的有效性和体积结果发现风险。
- 工具体在根部允许范围之外再次接触目标稳定面时，判定为目标曲面自交或意外回撞；工具体在有效深度范围内接触非目标稳定面时，判定为相邻面干涉，并在中文错误中报告最近接触距离、干涉面数量和稳定面 ID 摘要。
- 曲面通孔根据点击位置真实内法线得到的局部壁厚放行预计出口附近的正常接触；曲面盲孔不会使用这一出口豁免，因此圆形轮廓边缘提前撞到内部孔、筋位或其他面时仍会被拒绝。
- 成功结果新增 `interferenceCheckPassed`、自交/相邻面标记、干涉面数量与 ID、最近干涉距离，以及接触面和接触采样数量。平面特征返回“不适用”诊断，不改变既有平面执行行为。
- 干涉检查失败会抛出包含“曲面作用区域干涉检查未通过”的中文错误。Store 保留修改前 CAD、当前稳定面选择、真实曲面 UV 和三维工具预览，并把预览生命周期标记为 `blocked`；视口使用橙色工具体和“检测到曲面干涉，已阻止执行”风险卡片，用户可以清除预览或调整半径、深度和点击位置后重试。
- 普通 Worker 错误仍使用 `failed` 红色状态，避免把裁剪边界、曲率、壁厚、文件或环境错误错误归类为已经确认的几何干涉。
- 成功中文摘要显示检查的接触面数和接触采样数，并明确说明未发现目标曲面自交或非目标面干涉。

### 27. 曲面局部特征参数历史对比（已实现）

- OpenCascade Worker 只为非平面稳定面上的圆形凸台和圆孔保存 `curvedDiagnostics`，内容包括最大绝对曲率、最小曲率半径、曲率比、局部壁厚、剩余壁厚、通孔状态、自交与相邻面干涉、干涉稳定面 ID、最近干涉距离，以及接触面和接触采样数量；平面或非圆形特征不会误写曲面诊断。
- 参数化整模重建不直接复用历史诊断，而是在每次安全重放后根据当前 OpenCascade 几何重新计算；旧记录没有 `curvedDiagnostics` 时会自动补齐，旧记录包含过期或错误诊断时会被当前结果替换。
- 每次提交模型版本时深拷贝当前曲面特征诊断，并使用“创建修订 + 零件 ID + 操作类型”形成跨参数化重放的特征标识，避免后续 CAD 结果原地变化污染历史快照。
- 版本历史的中文差异界面新增“曲面局部特征变化”，支持识别新增、删除和修改，并逐项显示工具半径/直径、作用深度、中文曲面类型、曲率、壁厚、通孔、干涉稳定面和接触采样的前后值。
- 老版本没有曲面特征快照时按空列表兼容，不影响原有参数、开孔模式和通用开孔差异对比。

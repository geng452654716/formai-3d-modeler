# FormAI — AI 三维建模

FormAI 是面向 macOS 和三维打印的 AI 参数化建模软件。用户可以提供自然语言指令、参考图片、关键尺寸和电子元件数据，由 Codex 规划建模修改，再由本机 OpenCascade/CadQuery 内核生成和校验精确实体。

## 产品重点

- 通过自然语言和参考图片创建基础模型；
- 以毫米为统一单位进行一比一精确建模；
- 支持内部掏空、壁厚、装配间隙、开孔、圆角、倒角和电子元件包络；
- 支持对象、部件和局部区域选择后向 Codex 下达修改指令；
- 自动执行 AI 修改，并在每次修改前创建可回滚版本；
- 面向 Bambu Lab P1S，导出 STL、STEP 和几何 3MF；
- 首发平台为 macOS，不依赖 Fusion 360。

详细需求见 [产品规格](docs/product-spec.md)，技术方案见 [架构设计](docs/architecture.md)，当前交付边界见 [实施计划](docs/implementation-plan.md)。

## 当前已经实现

- React Three Fiber 三维视口，以及视角旋转、缩放、平移和对象选择；
- 默认装配视图，以及明确的“装配视图 / 拆分视图”切换；
- “新建模型”画布重置和版本历史初始化；
- PNG、JPEG、WebP 图片导入、拍照入口、双点真实尺寸标定和毫米/像素比例计算；
- 通过本机 Codex `--image` 逐张识别物体、尺寸参数、USB-C/按钮/LED/排针等接口，并在照片上叠加中文标记；
- 识别完成后可切换到“编辑接口”模式，直接在已标定照片上选择和拖动接口框、拖动右下角调整宽高，并可恢复本次 Codex 原始识别值；二维编辑会同步更新毫米尺寸、横向偏移和底部偏移，并使旧联合标定失效；
- 同一项目可管理正面、背面、左侧、右侧、顶部、底部和透视照片；每个视角独立保存双点标定、识别结果和接口观测；
- 至少两个已标定视角可执行联合标定：使用毫米/像素比例中位数融合尺度，报告最大相对偏差，并按稳定 ID 或类型与尺寸形成“已匹配 / 待确认”接口对应；
- 联合结果提供全中文接口人工复核：可确认同一接口、把错误候选拆为独立接口、忽略误识别，并修正 ID、类型、所在面、尺寸、照片位置、物理偏移、是否开孔和开孔轮廓；待确认项未处理完时禁止直接生成模型；
- 多视角元数据和联合结果保存到本机应用存储；图片二进制不写入该存储，重新识别时需要重新选择原图；
- 多视角尺寸估算按参数取中位数后写入模型；所有经过人工复核、需要开孔且可确定正交接口面的观测均可进入通用精确开孔协议，支持正面、背面、左侧、右侧、顶部和底部，以及圆孔、矩形孔、圆角矩形孔和槽孔；顶部开孔切上盖，其余五面切主体；
- 新生成的照片开孔保存“接口面水平中心 + 底边”的毫米定位锚点；外壳尺寸变化时自动重新计算开孔中心坐标，接口宽高、水平偏移和底部偏移保持毫米值，不按比例缩放；
- OpenCascade 封闭 CAD 实体的平面拆件、自动补面、有效性和体积守恒验证，并已接入桌面按钮、装配/拆分视图及拆件 STL/STEP 导出；
- 圆柱定位销、D 形防转销、燕尾榫、球头/球窝和磁铁孔已由 OpenCascade 精确布尔写入拆件实体；PLA/PETG 可拆悬臂卡扣、M2/M2.5/M3 螺丝柱、打印友好圆脊近似内外螺纹，以及 M2/M2.5/M3 ISO 公制粗牙 60° 内外螺纹均已实体化；
- 精确内核自动检测两侧可附着位置，并检查封闭性、悬浮 Solid、至少 1.20 毫米的连接结构设计壁厚、0.10–1.00 毫米装配间隙和公母干涉；
- 支持用户上传任意文件名和模型类别的 STL，完成拓扑诊断、退化/重复三角面清理、简单平面孔洞自动修复、实体校验、X/Y/Z 平面拆件、视口预览和两侧 STEP/STL 导出；
- 自动保留原始上传 STL，并在发生修复时另外生成“修复后工作 STL”；界面展示修洞数、补面三角形、修复前后开放边和连通网格数；
- 上传 STL 激活后，场景树、项目标题、右侧模型信息、候选连接位置和导出菜单均由上传模型的真实文件名、包围盒与拆件结果驱动，不使用 ESP32 或固定外壳结构；
- 支持对任意封闭 STEP/STL 执行全局壁厚“表面法向射线采样估算”，按危险（<0.80 毫米）、偏薄（0.80–1.20 毫米）、建议（1.20–2.00 毫米）和充足（≥2.00 毫米）分级，并在三维视口显示红/橙/黄/绿热力图、覆盖率、最薄值和 5% 分位；
- 热力图支持点击局部区域或一键定位最薄处，并把零件、原始毫米坐标、风险等级和估算壁厚附加到下一条 Codex 指令；CAD 参数化修改完成后会自动重新采样并报告该区域修改前后的壁厚变化；
- 上传任意封闭单 Solid STL 后，可在壁厚热力图中选择局部采样点，沿选中表面法向执行“圆形凸台加厚”或“圆孔切除”；修改后自动重新执行“表面法向射线采样估算”；
- 上传 STL 的连续局部编辑读取 `imported-model-result.json` 指向的当前工作文件，不会每次回到原始 STL；原始上传文件始终保留，STEP/STL 工作文件及结果清单采用带备份回滚的批量替换；
- 局部修改结果会经过 OpenCascade 有效性、封闭性、单一 Solid、体积方向和 STL 再导入检查；
- 当前电子元件保护壳模板：参考元件包络、掏空主体、接口开孔、定位导轨和独立上盖；
- ESP32-S3 N16R8 参数仅保留为 `modeling/examples/` 中的示例 Demo，不属于通用内核协议；
- 壁厚、底厚、圆角、倒角、间隙和接口尺寸的参数化修改；
- 撤销、重做、版本记录和本机版本快照；可打开全中文“版本历史与对比”面板，选择任意历史版本作为基准，查看全部参数增减量、模板/自定义开孔模式变化，以及通用开孔的新增、删除和字段修改，并恢复到任意参数化版本；带本机快照的历史版本还可与当前精确 CAD 执行蓝/橙半透明重叠、按真实毫米包围盒并排显示，或计算 OpenCascade 精确双向布尔差集，以绿色显示新增区域、红色显示删除区域并报告体积；每个参数化 CAD 零件还会记录“几何签名匹配第一版”的稳定面 ID，并在版本实体对比中统计共享、新增和消失编号；
- 参数化精确 CAD 会逐 OpenCascade 面独立三角化，并按稳定面顺序拼接成专用选择 STL；面映射文件为每个稳定面记录连续的 `triangleIndex` 区间，选择 STL 与面向用户的导出 STL 相互独立；
- 三维视口支持点击选择单个稳定 CAD 面，以及框选当前视角可见的多个稳定 CAD 面；框选会先判断屏幕空间三角形与矩形是否相交，再通过摄像机射线的第一命中过滤被遮挡的背面；
- 选择上下文会将装配视图或拆分视图中的命中位置逆变换回零件原始毫米坐标，并同步逆变换外法线；稳定面 ID、零件包围盒、摄像机矩阵、尺寸上下文和局部截图会一起发送给 Codex，截图通过 PNG 附件传输而不是写入文本；
- 点击任意稳定 CAD 面后，会按当前 `revision + partId + stableFaceId + triangleIndex` 调用 OpenCascade 重新定位 STEP 真实裁剪面，将选择网格命中点投影为真实曲面点，解析考虑面方向的外法线、曲面 UV 和 UV 边界，并回传投影距离与法线点积；异步旧结果不会覆盖后续选择，失败时只保留网格预览且不会伪造精确 UV；
- 点击单个稳定 CAD 平面后，可用全中文指令沿 OpenCascade 重新定位面的真实外法线增加圆形或旋转矩形凸台、切除圆孔/旋转矩形孔/槽孔，也可将整个平面向外拉伸或向内偏移；Codex 登录时只返回受限结构化计划，`partId + stableFaceId` 必须与当前点击选择完全一致，中心和法线始终取自已验证的选择上下文；Codex 不可用时才使用确定性中文尺寸解析器；
- 点击圆柱面等非平面稳定 CAD 面后，可基于当前修订中 OpenCascade 精确解析的真实三维点、外法线、UV 和单位 U 切向，沿真实外法线生成受限圆形或矩形凸台，或沿真实内法线切除圆孔、矩形孔和槽孔；曲面矩形与槽孔都在真实 UV 点击位置建立切平面，`0°` 沿真实 U 切向，正角度围绕真实外法线旋转。矩形使用半对角线、槽孔使用总长度一半作为曲率、裁剪边界、壁厚、自交和相邻面干涉检查的保守包络半径。该能力仍是点击位置切平面的安全近似，不是沿曲面贴合或测地线轮廓；盲孔要求至少保留 1.20 毫米壁厚，通孔只放行局部壁厚附近的正常出口接触，修改记录可参与参数化安全重放；
- 曲面圆形、矩形或受限槽孔特征在自动调用 OpenCascade Worker 前，会在目标零件上显示绑定当前 `revision + partId + stableFaceId + surfaceUv` 的半透明三维预览：凸台沿真实外法线显示，孔和槽孔沿真实内法线显示，矩形与槽孔预览包含实际宽高或宽长、深度和旋转角；预览具有“准备自动执行 / 正在自动执行 / 检测到曲面干涉，已阻止执行 / 执行失败，预览已保留”中文状态，成功后清除；干涉风险使用橙色预览并在布尔和文件写回前阻止执行，普通失败则保留原预览供调整尺寸重试，重新选择、模型修订变化或参数修改后自动失效；
- 点击平面所属的单条稳定 CAD 边后，可用全中文指令执行 0.20–50.00 毫米的单边圆角或等距倒角；目标由 `partId + stableFaceId + stableEdgeId` 共同限定，Codex 不允许切换目标或输出任意代码，OpenCascade Worker 会重新定位真实边并精确复核点击距离；第一版仅支持 `PLANE` 所属单边；
- 稳定 CAD 局部修改完成后会重新导出 STEP/STL/通用多零件 3MF，重新生成面描述、面内边描述、几何签名匹配摘要、选择 STL 和面三角映射，并把记录写入 `generation-result.json` 的 `localFeatures[]`；后续参数化整模重建会从基础参数化实体开始按记录顺序安全重放，并使用修改前 `targetFace` / `targetEdge` 几何签名快照辅助重新定位；
- 曲面圆形、矩形和受限槽孔特征会把曲率、局部壁厚、剩余壁厚、通孔状态、自交与相邻面干涉、干涉稳定面及接触采样诊断持久化到局部特征记录；参数化重放会按当前几何重新计算，版本历史可用全中文界面对比特征新增、删除，以及圆形直径、矩形宽高、槽孔宽长、旋转角、真实 U 切向、作用深度和诊断字段变化，并兼容旧版本缺少诊断快照或真实 U 切向的记录；
- CadQuery + OpenCascade 精确实体自动重建；
- 实体有效性、体积、包围盒和 P1S 成型尺寸检查；
- “OpenCascade 精确实体”和“快速参数预览”切换；
- 分件 STL、STEP，以及包含上下壳两个对象的标准几何 3MF 导出；
- Tauri 2 macOS 桌面壳和中文原生菜单；
- Rust 到 Python CAD Worker 的本机进程通信；
- 复用本机 Codex 登录状态的 Codex CLI 指令适配器；
- “修改前快照 → Codex 参数规划 → OpenCascade 重建 → 实体校验 → 提交版本”的自动执行闭环；
- Codex 不可用时自动回退到本地中文规则解析；
- 拆件策略、圆柱销、D 形销、燕尾榫、球头、磁铁孔、卡扣、螺丝柱和内外螺纹的制造方案规划器。

当前的 3MF 可由 Bambu Studio 打开，但还没有写入 Bambu Studio 专有的切片配置、耗材和支撑工程元数据。

新生成的照片精确开孔会保存底边定位锚点，并在外壳长、宽、高变化时按当前接口面尺寸重新计算中心坐标；接口宽高和物理偏移不会按比例缩放。老项目如果没有定位锚点元数据，仍按原固定毫米坐标兼容加载，界面会提示重新复核。单张透视照片不能单独确定精确接口面；打印前仍应使用卡尺复核孔位和孔径。边缘余量、孔间距、实体相交、有效性和单一 Solid 检查只提供几何与制造保护，不等同于受力、疲劳或有限元分析。

## 尚未实现

- 上传 STL 不套用参数化 CAD 的稳定面协议，当前使用独立的网格区域与壁厚离散采样协议；其任意自由曲面重建、多 Solid 局部编辑、复杂非流形/自相交修复和完整撤销/重做仍未实现，第一版仅支持封闭单 Solid 上的圆形凸台加厚和圆孔切除；
- 移动、旋转、缩放操作手柄和颜色调整；
- 顶点、边、面手工编辑；
- 稳定面以及所属稳定面内的稳定边都使用“几何签名匹配第一版”，不是 OpenCascade 原生永久拓扑命名；大幅拓扑变化、对称面或布尔重建仍可能使编号变化。`triangleIndex` 只对本次生成的选择网格有效，任何局部修改后都会立即失效，稳定边也必须重新选择；整模重建只会在稳定面、稳定边、记录中心、外法线和尺寸全部通过安全校验时重放 `localFeatures[]`，任一记录失败会拒绝整次重建并保留最后有效模型；
- Codex App Server 持久会话；
- 发布版内置 Python/CadQuery 运行环境。目前开发模式使用 `modeling/.venv`；
- 面向复杂破损手办网格的非流形、自相交、分叉/嵌套边界、非共面破面和细小孤岛修复，以及曲面拆件、基于曲率/受力的结构薄弱分析、打印方向评估和自动连接器布置；
- 从照片直接生成任意物体三维网格；当前照片能力只做逐张识别、尺寸标定、尺度中位数融合和接口候选匹配；
- Bambu Studio 专有工程 3MF。

## 本地运行

先准备能够导入 CadQuery 的 Python 环境。当前项目默认使用：

```bash
modeling/.venv/bin/python
```

生成初始模型并运行网页开发模式：

```bash
pnpm install
pnpm run model:generate
pnpm run dev
```

运行 macOS 桌面应用：

```bash
pnpm run desktop:dev
```

桌面端默认复用本机 Codex 桌面应用或 Codex CLI 的现有登录配置，不会读取或显示认证密钥。

### 30. 非平面矩形凸台与矩形孔安全执行（已实现）

- 非平面稳定面现已允许 `add-rectangle` 和 `cut-rectangle`。矩形位于当前修订真实 UV 点击位置的切平面，`rotationDeg=0` 沿 OpenCascade 真实 U 切向，正角度围绕真实外法线；凸台沿外法线加料，矩形孔沿内法线切削。
- Worker 会在当前 STEP、稳定面和 UV 上重新计算真实 U 切向，并校验请求上下文中的切向没有过期、反向或退化；Codex 受限 JSON 不包含可编辑切向字段，不能生成或改写该方向。
- 曲面矩形以 `sqrt(width² + height²) / 2` 作为保守包络半径，统一参与曲率比、裁剪边界、局部壁厚、剩余壁厚、通孔出口、自交和相邻稳定面干涉检查，不会只检查矩形四角或低估作用范围。
- Three.js 使用与 OpenCascade 相同的真实 U 切向、真实外法线和旋转角构建半透明矩形凸台或矩形孔预览；`localFeatures[]` 持久化宽、高、深度、旋转角、真实 U 切向和曲面诊断。
- 参数化安全重放会按当前 OpenCascade 几何重新计算 U 切向和半对角线诊断；旧记录没有 U 切向时可安全补写，携带反向或不一致切向时会在修改模型前拒绝。
- 全中文版本历史支持“曲面矩形凸台”和“曲面矩形孔”，并比较轮廓宽度、矩形高度、旋转角、曲面 U 切向、作用深度及曲率、壁厚、通孔和干涉诊断。
